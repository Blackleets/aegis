# AEGIS Brand System

## Goal
Hacer que AEGIS se perciba como un producto propio, no como un fork visual.

## Principios
- **Operador, no templo**: evitar simbología prestada o demasiado reconocible.
- **Geometría táctica**: hexágonos, triángulos, retículas y marcas de rumbo.
- **Señal verificada**: lenguaje de confianza, no marketing genérico.
- **Baja latencia visual**: identidad fuerte sin overlays pesados.

## Voz
- Preferir: `Verified`, `Command Surface`, `Signal`, `Posture`, `Fusion`, `Intel`, `Telemetry`.
- Evitar: nombres heredados del proyecto original y etiquetas vagas tipo `Prime Shell` si no aportan identidad propia.

## Iconografía
- El símbolo base de AEGIS debe ser **geométrico y propio**.
- Usar combinación de:
  - marco hexagonal
  - núcleo triangular
  - línea de base / horizonte
  - acento dorado + cian
- Evitar ojo/horus/falcon/glyphs reconocibles de terceros.

## Paleta núcleo
- Fondo: `#03070D` / `#07111A`
- Oro AEGIS: `#D4AF37`
- Oro alto: `#F5D76E`
- Cian señal: `#00E5FF`
- Cian profundo: `#00A9D1`
- Texto humo: `#7AA6B8`

## Primera pasada ejecutada
- metadata pública renombrada a AEGIS
- share copy renombrado a AEGIS
- AI/operator copy renombrado a AEGIS
- favicon SVG reemplazado por símbolo geométrico propio
- manifests/robots alineados con `aegis.blackleets.dev`

## Próximas pasadas recomendadas
1. Reemplazar residuos `WorldWatch` restantes en prompts internos y componentes secundarios.
2. Sustituir iconografía Lucide visible de cabeceras principales por 2-3 glyphs SVG propios de AEGIS.
3. Crear `og-image` propia y sustituir PNGs heredados del paquete favicon actual.
4. Limpiar assets públicos no usados (`osiris-icon.png`, `casaos-icon.png`, `next.svg`, `vercel.svg`, etc.) si ya no están referenciados.
